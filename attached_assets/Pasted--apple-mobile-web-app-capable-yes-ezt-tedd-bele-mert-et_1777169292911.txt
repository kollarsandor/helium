- **`apple-mobile-web-app-capable=yes`** — ezt tedd bele, mert ettől indul a Home Screenről külön appként, Safari UI nélkül; enélkül mindig böngészőérzet marad iPhone-on 
```html
<meta name="apple-mobile-web-app-capable" content="yes">
```

- **`apple-mobile-web-app-status-bar-style`** — ezt tedd bele, mert standalone módban ezzel lesz natívabb a felső sáv viselkedése; nálad a `black-translucent` jó, ha a designod már most ilyen lebegős/üveges[1]
```html
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
```

- **`apple-mobile-web-app-title`** — ezt tedd bele, mert a főképernyőn ez lesz az app neve, nem a fájlnév vagy egy véletlen title 
```html
<meta name="apple-mobile-web-app-title" content="Chat">
```

- **`apple-touch-icon`** — ezt tedd bele, mert iPhone Home Screen ikonhoz ez a biztos Apple-es út; ettől lesz appikon-érzet, nem screenshotos webclip 
```html
<link rel="apple-touch-icon" href="/static/icon-180.png">
<link rel="apple-touch-icon" sizes="152x152" href="/static/icon-152.png">
<link rel="apple-touch-icon" sizes="167x167" href="/static/icon-167.png">
<link rel="apple-touch-icon" sizes="180x180" href="/static/icon-180.png">
```

- **`apple-touch-startup-image`** — ezt is rakd bele, ha tényleg a lehető legnatívabb launch-érzet kell, mert Apple külön írja, hogy ezzel saját indulóképet adhatsz a web appnak 
```html
<link rel="apple-touch-startup-image" href="/static/launch.png">
```

- **`viewport-fit=cover` + safe area** — ezt tedd bele, mert notchos iPhone-on ettől tudsz teljes képernyőn menni úgy, hogy közben nem csúszik bele a tartalom a szenzorszigetbe és a home indicatorba[1]
```html
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover, interactive-widget=resizes-content">
<style>
:root{
  --sat: env(safe-area-inset-top, 0px);
  --sar: env(safe-area-inset-right, 0px);
  --sab: env(safe-area-inset-bottom, 0px);
  --sal: env(safe-area-inset-left, 0px);
}
.header{ padding-top: var(--sat); }
.footer{ padding-bottom: var(--sab); }
.screen{
  padding-left: var(--sal);
  padding-right: var(--sar);
}
</style>
```

- **`interactive-widget=resizes-content`** — ezt tedd bele, mert iPhone billentyűzetnél sokkal rendezettebben viselkedik az alsó inputterület és nem annyira “webes” a layout-törés[1]
```html
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover, interactive-widget=resizes-content">
```

- **`min-height: 100dvh`** — ezt tedd bele, mert iPhone-on a dinamikus viewport miatt ez sokkal natívabb fullscreen viselkedést ad, mint a sima régi `100vh`[1]
```css
html,body{
  height:100%;
}
#app{
  height:100vh;
  min-height:100dvh;
}
```

- **`window.visualViewport`** — ezt használd, mert billentyűzetnél ettől tudod pontosan követni a látható viewportot, és az inputbar úgy marad lent, mint egy natív chat appban[1]
```html
<script>
const bar = document.getElementById('inputbar');

function syncKeyboard(){
  const vv = window.visualViewport;
  if(!vv || !bar) return;
  const keyboardInset = Math.max(0, window.innerHeight - vv.height - vv.offsetTop);
  bar.style.transform = `translateY(-${keyboardInset}px)`;
}

if(window.visualViewport){
  window.visualViewport.addEventListener('resize', syncKeyboard, {passive:true});
  window.visualViewport.addEventListener('scroll', syncKeyboard, {passive:true});
  syncKeyboard();
}
</script>
```

- **`-webkit-overflow-scrolling: touch` + egyetlen fő scroll-régió** — ezt tedd bele, mert iPhone-on ettől lesz a görgetés közelebb a natívhoz; ne legyen öt egymásba ágyazott scrollozó doboz[1]
```css
#scroll-area{
  position:absolute;
  inset:0;
  overflow-y:auto;
  overflow-x:hidden;
  -webkit-overflow-scrolling: touch;
  overscroll-behavior-y: contain;
  touch-action: pan-y;
}
```

- **`-webkit-tap-highlight-color: transparent`** — ezt tedd bele, mert eltünteti a tipikus iOS Safari tap-highlight villanást, ami rögtön lebuktatja, hogy web[1]
```css
*{
  -webkit-tap-highlight-color: transparent;
}
```

- **natív kinézetű form/reset** — ezt tedd bele gombokra, keresőkre, inputokra, mert különben a WebKit saját random UI-elemei beleszólnak a kinézetbe [user query]
```css
button,
input,
textarea,
select{
  -webkit-appearance:none;
  appearance:none;
  border:none;
  background:none;
  font:inherit;
  color:inherit;
}
```

- **glass rétegek `backdrop-filter`-rel** — ezt tedd bele overlayre, sidebarra, alsó sávra, mert az iOS saját felületeinek egyik legerősebb natív érzetű vizuális jele a blur + saturate[1]
```css
.glass{
  background: rgba(20,20,24,.72);
  backdrop-filter: blur(40px) saturate(160%);
  -webkit-backdrop-filter: blur(40px) saturate(160%);
  border: .5px solid rgba(255,255,255,.08);
}
```

- **rendszerfont stack** — ezt használd, mert ha az a cél, hogy még iPhone-on se lógjon ki, akkor a rendszer tipográfia a legerősebb álca[1]
```css
html,body,input,button,textarea{
  font-family:-apple-system,BlinkMacSystemFont,"SF Pro Text","SF Pro Display","Helvetica Neue",sans-serif;
}
```

- **16px input fontméret** — ezt tartsd meg, mert iPhone Safari hajlamos zoomolni kisebb input-fontnál, és ettől azonnal webesnek érződik[1]
```css
textarea,
input{
  font-size:16px;
}
```

- **standalone ellenőrzés** — ezt használd, mert így tudsz külön finomhangolást adni csak akkor, amikor már tényleg Home Screen appként fut 
```html
<script>
const isStandalone = window.navigator.standalone === true;
```

- **standalone-specifikus finomhangolás** — ezt használd, hogy csak telepített állapotban legyen teljes natív shell

```css
  background:#000;

  padding-top:0;
```

- **touch callout és selection tiltás csak ott, ahol kell** — ezt használd ikonokra, toolbarra, gesture-elemekre, mert ezeknél zavaró a Safari saját kijelölős/callout viselkedése [user query]

```css
#inputbar{
```

- **amit NE rakj bele: `UIWebView`** — ezt hagyd ki teljesen, mert elavult és App Store / modern iOS szempontból rossz irány[2]

- **amit NE rakj bele: `cache.manifest` / AppCache** — ezt hagyd ki teljesen, mert elavult offline megoldás, ma már service worker a jó út[3]

- **amit NE rakj bele: Web Bluetoothra épülő iPhone PWA logika** — ezt hagyd ki, mert iOS Safari oldalon erre nem lehet stabilan natív-álcás megoldást építeni[4][5]

- **amit NE rakj bele: Vibration API-ra épített haptika** — ezt hagyd ki, mert iPhone Safarin ez nem egy megbízható út; ha valódi haptika kell, az már natív wrapper/bridge irány[6][7]

- **amit NE rakj bele: biometria/Keychain/HealthKit “sima HTML-ből”** — ezt hagyd ki, mert ezekhez már natív bridge kell, nem tiszta egyfájlos PWA[8][9]

- **amit NE rakj bele: kinézetet szétverő agresszív reset** — ezt ne így csináld:
```css
*{
  all:unset;
}
```
